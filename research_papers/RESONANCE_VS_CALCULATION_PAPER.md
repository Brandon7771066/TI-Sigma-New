## 2. Defining the Two Modes

### 2.1 Calculation

**Definition:** Deterministic transformation of input data through algorithmic rules to produce output.

**Characteristics:**
| Property | Description |
|----------|-------------|
| Input-bound | Output determined entirely by input |
| Algorithmic | Follows specified rules |
| Reproducible | Same input → same output |
| Traceable | Each step can be audited |
| Limited | Cannot exceed information in input |

**Formula:**
```
Output = Algorithm(Input)
```

### 2.2 Resonance

**Definition:** Pattern recognition that enables retrieval of correlated information not present in the immediate input.

**Characteristics:**
| Property | Description |
|----------|-------------|
| Pattern-triggered | Recognition activates retrieval |
| Non-algorithmic | No fixed transformation rules |
| Contextual | Same pattern can retrieve different information |
| Non-traceable | Process is not step-by-step auditable |
| Unlimited | Can access information beyond input |

**Formula:**
```
Information = Resonance(Pattern → Recognition → Retrieval)
```
