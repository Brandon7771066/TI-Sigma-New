## Core Principle

**Publish the "what" and "how" selectively. Protect the "why" and unified system.**

The TI Framework's value lies in its unified theory. Individual insights are valuable for credibility, but the full system should remain proprietary until properly licensed or patented.

## Safe Publication Checklist

Before publishing ANY TI-derived content:

1. [ ] **Remove TI terminology** (GILE, LCC, CCC, i-cell, Myrion, etc.)
2. [ ] **Replace thresholds** with "empirically derived" language
3. [ ] **Generalize weights** (say "weighted" not "0.35")
4. [ ] **Add conventional citations** (Kahneman, Shannon, etc.)
5. [ ] **Check cross-references** to protected content
6. [ ] **Timestamp original** in Zenodo/GitHub before publication
7. [ ] **Review with IP attorney** if commercial value involved
